About Legate Studio

Legate Studio is a personal knowledge base built for the era of AI-assisted work — made by ThinxAI, a small team building infrastructure for how people think alongside AI.

Our Mission

Personal knowledge infrastructure for the AI era

The problem we're solving is simple to state and hard to fix: AI tools are stateless. Every conversation starts from scratch. Your Claude session doesn't remember the context from last week. Your AI assistant doesn't know about the project you've been thinking about for months.

The standard workaround is copy-pasting context into every prompt. That's a patch, not a solution. The real fix is a structured, persistent memory layer that AI tools can query directly — one that's built on your actual thinking, not just chat logs.

That's what Legate Studio is. A personal knowledge base where every note is a node, relationships form automatically, and the whole thing is queryable by any AI tool via MCP (Model Context Protocol). Not a note app with an AI chat widget bolted on — a knowledge base designed from the ground up to be an AI-usable memory substrate.

Why This Category Exists

The gap between note-taking and AI memory

Personal knowledge management tools have been around for decades. Roam Research, Obsidian, Notion, Bear — they're all good at storing and organizing notes. The problem is that none of them were designed to be read by AI tools in real-time.

Meanwhile, AI assistants have become genuinely useful for knowledge work — but their usefulness is bounded by what they know about you. A generic AI gives generic answers. An AI with access to your notes, your projects, your thinking patterns gives something closer to a knowledgeable collaborator.

The Model Context Protocol, developed by Anthropic, created a standard for exactly this kind of integration. MCP lets AI tools query external data sources during a conversation — turning a static knowledge base into a live memory layer.

Legate Studio was built to occupy this gap: a personal knowledge base that's organized enough to be useful to AI tools, and structured enough that those tools can reason over it meaningfully. Not a document dump, not a raw text store — a structured graph that AI can navigate.

What We Believe

Principles behind the product

Your data belongs to you

Your knowledge base is stored in your own SQLite database on a volume you can download. No platform lock-in, no proprietary formats. Your notes are markdown text in a standard database.

Capture should be frictionless

The hardest part of building a knowledge base is getting things into it consistently. Legate minimizes that friction — voice input, text paste, fast processing. Getting things in has to be easier than not getting things in.

Structure should emerge, not be imposed

You shouldn't have to organize your notes. The AI pipeline handles categorization, relationship mapping, and graph placement automatically. You capture — the structure builds itself.

AI integration should be architectural

MCP integration isn't a feature we added to a note app. It's built into the data model. The knowledge base was always designed to be queryable by AI tools — not adapted for it after the fact.

Pricing should be honest

The BYOK tier exists because we don't want to mark up AI costs. If you bring your own Gemini API key, you pay Google at their rates and us $0.99/month for the platform. No hidden margins, no bundled usage you didn't ask for.

Publishing is a power users want

Sharing specific notes while keeping the rest private is a real workflow. Researchers, developers, writers — they want to share some thinking, not all of it. Public profiles and per-note publishing exist because that distinction matters.
